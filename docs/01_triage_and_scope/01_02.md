# Task 02 - Correlate the scattered signals with Security Copilot

## Introduction

The incident graph shows the connections, but turning it into something you can explain to Vikram (the CISO) still takes work. This is exactly where standalone **Security Copilot** earns its place: as a reasoning layer that reads the incident — its alert, entities, and the underlying Sentinel telemetry — and tells the story in plain language.

## Description

You will use standalone Security Copilot to summarize the incident and **correlate the signals across layers into a single cross-domain narrative** — one chain, from initial access to execution.

## Success criteria

- Security Copilot produces **one incident narrative spanning identity, email/poisoning, AI, infrastructure, and endpoint** _(Validation: cross-domain narrative produced)_
- The narrative names the AI-layer involvement (the Refund Agent and its grounding source), not just the commodity identity/endpoint signals
- You have saved the narrative to your running notes

## Key tasks

### 01: Summarize the incident with Copilot

Get the machine-speed read of what happened.

<details markdown="block">
<summary><strong>Expand this section to view the solution</strong></summary>

1. You can work in either surface. In the Defender portal, open the incident and select **Copilot** to summarize it in context; or use standalone Copilot at [https://securitycopilot.microsoft.com](https://securitycopilot.microsoft.com){:target="_blank"} and reference the incident by ID.

2. In standalone Copilot, submit:

   ```text
   Summarize Microsoft Defender incident <INCIDENT_ID> for a SOC lead. Describe
   the contributing alert and its detection source, the MITRE tactics it covers,
   and the affected users, devices, and cloud/AI resources.
   ```

   Replace `<INCIDENT_ID>` with the numeric incident ID shown next to the title in the portal (for example, `210`).

   {: .note }
   > 📷 **Screenshot to capture:** the Copilot incident summary listing alerts, detection sources, and affected assets.

</details>

### 02: Correlate across layers into one narrative

Push Copilot from "list of signals" to "single chain."

<details markdown="block">
<summary><strong>Expand this section to view the solution</strong></summary>

1. Follow up in the same session so Copilot keeps context:

   ```text
   Treat the staged events in this incident as one coordinated attack rather than
   separate signals. Correlate them into a single cross-domain narrative ordered
   by time, spanning identity, email, the Zava Refund Agent (AI), the inference
   infrastructure (the sp-refund-agent-inference service principal), and the
   endpoint. For each step, name the entity involved and the layer.
   ```

2. Read the result critically. A good narrative connects: the **anonymous IP sign-in** (Tor exit node, classified as anonymizing infrastructure) for Mariya Petrova → a **phishing email** from a look-alike sender → a **poisoned document** in the Refund Policy Knowledge Base → **prompt-injection probing** of the Refund Agent → **anomalous infrastructure requests** by `sp-refund-agent-inference` → **script execution** on OPS-LT-0427.

3. If Copilot stops at the identity/endpoint signals and omits the AI layer, prompt it explicitly:

   ```text
   Include the AI layer: what activity targeted the Zava Refund Agent and its
   grounding source, and how does it connect to the identity compromise?
   ```

   {: .warning }
   > Security Copilot responses are non-deterministic. The narrative's wording will vary between runs; what must be present is the **cross-layer chain including the AI stage**. If a run misses it, re-prompt as shown — this is itself a lesson in supervising AI-driven SecOps.

</details>

### 03: Capture the narrative

Lock in the output you'll build on.

<details markdown="block">
<summary><strong>Expand this section to view the solution</strong></summary>

1. Use **Pin** in Copilot (or copy the text) to save the cross-domain narrative to your running notes / pinboard.

2. In one sentence of your own, write the chain as you now understand it. You'll test and refine this sentence against the telemetry in Exercise 02.

</details>

Congratulations! You've turned the incident graph of correlated signals into a single cross-domain narrative — explicitly including the AI layer that commodity triage tends to drop — and saved it. Next, you'll pull the concrete entities and an initial timeline out of this narrative so reconstruction has firm anchors.
