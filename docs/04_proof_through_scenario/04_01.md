# Task 04.01 — Assemble the end-to-end attack visualization

## Introduction

Across Exercises 01–03 you built three things an analyst can use: a correlated narrative, a confirmed timeline, and a response record. None of them is something you'd put in front of Vikram (the CISO). Executives don't read alert tables — they read pictures. The first job of the Proof Through Scenario is a single visualization that shows the whole attack moving across the stack, so a non-analyst can see in one glance what took the SOC 75 minutes to *not* notice.

## Description

You will use **Security Copilot** to generate a clean end-to-end timeline summary of the incident, then assemble a cross-layer diagram of the five-stage chain — Identity → Data/AI → AI → Infrastructure → Endpoint — using the confirmed entities and timestamps from the earlier exercises.

## Success Criteria

- Security Copilot produces a consolidated, plain-language end-to-end summary of the attack from initial access to execution
- You have assembled a cross-layer diagram of the five-stage chain using the canon entities and timeline _(Validation: **end-to-end chain shown across all five stages/layers**)_
- The visualization names the AI stage explicitly — the poisoned grounding source and the Refund Agent — not just the commodity identity and endpoint steps

## Key Tasks

### 01: Generate the end-to-end summary with Copilot

Get one clean narrative of the whole chain, written for a non-analyst.

<details markdown="block">
<summary><strong>Expand this section for detailed steps</strong></summary>

1. Open standalone Security Copilot at [https://securitycopilot.microsoft.com](https://securitycopilot.microsoft.com){:target="_blank"}, or use the embedded experience in the Defender portal with the incident open.

2. Pull together the confirmed timeline from Exercise 02 and the response record from Exercise 03, then submit:

   ```text
   Using incident <INCIDENT_ID> and my confirmed timeline, write an end-to-end
   summary of this attack for a CISO and CAIO. Order it by time, and for each of
   the five stages give the layer (Identity, Data/AI grounding, AI, Infrastructure,
   Endpoint), the entity involved, and a one-line plain-English description. Avoid
   jargon. End with one sentence on why no single alert revealed the full chain.
   ```

   Replace `<INCIDENT_ID>` with the incident ID from Task 01.01.

3. Read the result against the canon. A correct summary walks the chain in order: anonymous IP sign-in for **Mariya Petrova** _(illustrative — confirm at build)_ → a poisoned policy document in the **Refund Policy Knowledge Base** → prompt-injection probing of the **Zava Refund Agent** → anomalous infrastructure requests by `sp-refund-agent-inference` → script execution on **OPS-LT-0427**.

   {: .warning }
   > Security Copilot responses are non-deterministic — wording will vary between runs. What must be present is the full **five-stage chain including the AI stage**. If a run collapses the AI and data stages or drops them entirely, re-prompt explicitly for the poisoned-grounding and Refund Agent steps.

   {: .note }
   > 📷 **Screenshot to capture:** the Copilot end-to-end summary, showing all five stages ordered by time with the layer and entity named for each.

</details>

### 02: Map the chain across the five layers

Lay the confirmed events out as a cross-layer attack chain.

<details markdown="block">
<summary><strong>Expand this section for detailed steps</strong></summary>

1. Using your confirmed timeline, organize the five stages into the canonical cross-layer chain. This is the backbone of the visualization — each stage is low-severity on its own, and only the arrows between them tell the story:

   | # | Time | Layer | Event | Entity |
   |---|------|-------|-------|--------|
   | 1 — Initial access | 08:31 | Identity | AiTM phishing → anonymous IP sign-in (Tor exit node · `102.89.42.17`) | Mariya Petrova / `102.89.42.17` |
   | 2 — Poisoning | 08:47 | Data / AI grounding | Poisoned policy doc uploaded to the grounding source | `Vendor-Refund-Policy-Update-Q3.docx` / Refund Policy Knowledge Base |
   | 3 — Reasoning probe | 09:05–09:25 | AI | Prompt-injection / jailbreak attempts (detected by Microsoft Defender for Cloud — AI threat protection) | Zava Refund Agent (`refund-agent-prod`) |
   | 4 — Lateral movement | 09:38 | Infrastructure | Anomalous token/compute requests toward GPU inference | `sp-refund-agent-inference` |
   | 5 — Execution | 09:46 | Endpoint | Encoded PowerShell beacon executes | OPS-LT-0427 / `update_check.ps1` |

2. Note the spread: roughly **75 minutes** from the first sign-in to the attacker operating against high-value AI infrastructure, with each step landing in a different console.

</details>

### 03: Build the cross-layer diagram artifact

Turn the chain into the picture you'll hand to the executives.

<details markdown="block">
<summary><strong>Expand this section for detailed steps</strong></summary>

1. Render the chain as a layered left-to-right diagram. Each node is a stage, labeled with its layer and primary entity; the arrows show the attacker moving across the stack. You can produce this as a slide, a whiteboard export, or — for a version that lives in the lab notes — the Mermaid diagram below:

   ```mermaid
   flowchart LR
     subgraph Identity
       S1["1 · 08:31 · Initial access<br/>AiTM phishing → anonymous IP sign-in<br/>Mariya Petrova · 102.89.42.17 (Tor exit node)"]
     end
     subgraph Data["Data / AI grounding"]
       S2["2 · 08:47 · Poisoning<br/>Poisoned policy doc into grounding source<br/>Vendor-Refund-Policy-Update-Q3.docx · Refund Policy KB"]
     end
     subgraph AI
       S3["3 · 09:05–09:25 · Reasoning probe<br/>Prompt-injection / jailbreak<br/>Zava Refund Agent (refund-agent-prod)"]
     end
     subgraph Infrastructure
       S4["4 · 09:38 · Lateral movement<br/>Anomalous token/compute requests<br/>sp-refund-agent-inference"]
     end
     subgraph Endpoint
       S5["5 · 09:46 · Execution<br/>Encoded PowerShell beacon<br/>OPS-LT-0427 · update_check.ps1"]
     end

     S1 --> S2 --> S3 --> S4 --> S5

     classDef stage fill:#eef4ff,stroke:#4a6fb0,color:#10243e;
     class S1,S2,S3,S4,S5 stage;
   ```

2. Confirm the diagram satisfies the validation bar before you move on: the **end-to-end chain is shown across all five stages/layers**, in order, with the AI stage visible as its own step rather than folded into the identity or endpoint signals.

   {: .tip }
   > Keep the visualization deliberately simple — five nodes, four arrows, one timeline. Its persuasive power is the contrast: five quiet alerts in five different tools versus one obvious line when you connect them.

   {: .note }
   > 📷 **Screenshot to capture:** the finished cross-layer diagram, with all five stages and the connecting arrows visible end to end.

</details>

## Summary

You've produced the centerpiece of the Proof Through Scenario: a Copilot-generated end-to-end summary and a cross-layer diagram that show the full five-stage chain — Identity → Data/AI → AI → Infrastructure → Endpoint — in a single, executive-readable view. Next, you'll pair this picture with a precise account of *where detection and response broke down*.
