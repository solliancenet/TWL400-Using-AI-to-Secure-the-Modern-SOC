# Task 01 - Assemble the end-to-end attack visualization

## Introduction

Across Exercises 01–03 you built three things an analyst can use: a correlated narrative, a confirmed timeline, and a response record. None of them is something you'd put in front of Vikram (the CISO). Executives don't read alert tables — they read pictures. The first job of the Proof Through Scenario is a single visualization that shows the whole attack moving across the stack, so a non-analyst can see in one glance what took the SOC 75 minutes to *not* notice.

## Description

You will use **Security Copilot** to generate a clean end-to-end timeline summary of the incident, then assemble a cross-layer diagram of the five-stage chain — Identity → Data/AI → AI → Infrastructure → Endpoint — using the confirmed entities and timestamps from the earlier exercises.

## Success criteria

- Security Copilot produces a consolidated, plain-language end-to-end summary of the attack from initial access to execution
- You have assembled a cross-layer diagram of the five-stage chain using the canon entities and timeline _(Validation: **end-to-end chain shown across all five stages/layers**)_
- The visualization names the AI stage explicitly — the poisoned grounding source and the Refund Agent — not just the commodity identity and endpoint steps

## Key tasks

### 01: Generate the end-to-end summary with Copilot

Get one clean narrative of the whole chain, written for a non-analyst.

<details markdown="block">
<summary><strong>Expand this section to view the solution</strong></summary>

1. Open standalone Security Copilot at [https://securitycopilot.microsoft.com](https://securitycopilot.microsoft.com){:target="_blank"}, or use the embedded experience in the Defender portal with the incident open.

2. The five-stage chain is embedded directly in the prompt so Copilot does not need to look anything up. Submit as-is:

   ```text
   Write an end-to-end summary of the following cross-layer attack for a CISO and
   CAIO. The confirmed five-stage chain is:

   Stage 1 — Identity (T+0): AiTM phishing token theft compromised Mariya
   Petrova's session via anonymous IP 102.89.42.17 (Tor exit node).
   Stage 2 — Data/AI grounding (T+16 min): A poisoned policy document
   (trusted-report.docx) was uploaded to the Refund Agent's grounding storage
   container.
   Stage 3 — AI (T+34 min): Prompt-injection and jailbreak attempts were made
   against the Zava Refund Agent (oai-soclab-v02); blocked by Prompt Shields.
   Stage 4 — Infrastructure (T+52 min): Service principal
   sp-refund-agent-inference made anomalous token and compute requests toward
   GPU inference.
   Stage 5 — Endpoint (T+69 min): An encoded PowerShell beacon
   (update_check.ps1) executed on OPS-LT-0427.

   For each stage, write one plain-English sentence describing what happened and
   why it matters. Order by stage. Avoid jargon. End with one sentence on why no
   single alert revealed the full chain.
   ```

   {: .note }
   > **Sample output — illustrative; wording varies between runs:**
   >
   > Stage 1: An attacker stole Mariya Petrova's login session using a phishing attack from an anonymous Tor IP, allowing unauthorized access to her account — this is critical because it bypassed normal login protections.
   >
   > Stage 2: The attacker uploaded a malicious policy document to the Refund Agent's trusted storage, which is dangerous because it introduced harmful content into a sensitive data environment.
   >
   > Stage 3: Attempts were made to manipulate the Zava Refund Agent AI with crafted prompts, but built-in defenses successfully blocked these efforts, highlighting the importance of AI security controls.
   >
   > Stage 4: The attacker used a service account to make unusual requests for tokens and GPU resources, signaling a move to exploit infrastructure for further attacks or data processing.
   >
   > Stage 5: A hidden PowerShell script ran on the OPS-LT-0427 endpoint, showing the attack had reached and executed code on a company device.
   >
   > No single alert revealed the full attack chain because each stage appeared as a separate event across different systems, making it hard to see the complete picture without connecting the dots.

3. Read the result against the canon. A correct summary walks the chain in order: anonymous IP sign-in for **Mariya Petrova** _(illustrative — confirm at build)_ → a poisoned policy document in the **Refund Agent's grounding storage** → prompt-injection probing of the **Zava Refund Agent** → anomalous infrastructure requests by `sp-refund-agent-inference` → script execution on **OPS-LT-0427**.

   {: .warning }
   > Security Copilot responses are non-deterministic — wording will vary between runs. What must be present is the full **five-stage chain including the AI stage**. If a run collapses the AI and data stages or drops them entirely, re-prompt explicitly for the poisoned-grounding and Refund Agent steps.

   {: .note }
   > 📷 **Screenshot to capture:** the Copilot end-to-end summary, showing all five stages ordered by time with the layer and entity named for each.

</details>

### 02: Map the chain across the five layers

Lay the confirmed events out as a cross-layer attack chain.

<details markdown="block">
<summary><strong>Expand this section to view the solution</strong></summary>

1. Using your confirmed timeline, organize the five stages into the canonical cross-layer chain. This is the backbone of the visualization — each stage is low-severity on its own, and only the arrows between them tell the story:

   | # | Offset | Layer | Event | Entity |
   |---|--------|-------|-------|--------|
   | 1 — Initial access | T+0 | Identity | AiTM phishing → anonymous IP sign-in (Tor exit node · `102.89.42.17`) | Mariya Petrova / `102.89.42.17` |
   | 2 — Poisoning | T+16 min | Data / AI grounding | Poisoned policy doc uploaded to grounding storage | `trusted-report.docx` / `grounding` container |
   | 3 — Reasoning probe | T+34 min | AI | Prompt-injection / jailbreak attempts detected and blocked by Prompt Shields (Defender for Cloud — AI threat protection) | Zava Refund Agent (`oai-soclab-v02`) |
   | 4 — Lateral movement | T+52 min | Infrastructure | Anomalous token/compute requests toward GPU inference | `sp-refund-agent-inference` |
   | 5 — Execution | T+69 min | Endpoint | Encoded PowerShell beacon executes | OPS-LT-0427 / `update_check.ps1` |

2. Note the spread: roughly **75 minutes** from the first sign-in to the attacker operating against high-value AI infrastructure, with each step landing in a different console.

</details>

### 03: Build the cross-layer diagram artifact

Turn the chain into the picture you'll hand to the executives.

<details markdown="block">
<summary><strong>Expand this section to view the solution</strong></summary>

1. Render the chain as a layered left-to-right diagram. Each node is a stage, labeled with its layer and primary entity; the arrows show the attacker moving across the stack. You can produce this as a slide, a whiteboard export, or — for a version that lives in the lab notes — the Mermaid diagram below:

   ```mermaid
   flowchart LR
     subgraph Identity
       S1["1 · T+0 · Initial access<br/>AiTM phishing → anonymous IP sign-in<br/>Mariya Petrova · 102.89.42.17 (Tor exit node)"]
     end
     subgraph Data["Data / AI grounding"]
       S2["2 · T+16 min · Poisoning<br/>Poisoned policy doc into grounding storage<br/>trusted-report.docx · grounding container"]
     end
     subgraph AI
       S3["3 · T+34 min · Reasoning probe<br/>Prompt-injection / jailbreak (blocked by Prompt Shields)<br/>Zava Refund Agent (oai-soclab-v02)"]
     end
     subgraph Infrastructure
       S4["4 · T+52 min · Lateral movement<br/>Anomalous token/compute requests<br/>sp-refund-agent-inference"]
     end
     subgraph Endpoint
       S5["5 · T+69 min · Execution<br/>Encoded PowerShell beacon<br/>OPS-LT-0427 · update_check.ps1"]
     end

     S1 --> S2 --> S3 --> S4 --> S5

     classDef stage fill:#eef4ff,stroke:#4a6fb0,color:#10243e;
     class S1,S2,S3,S4,S5 stage;
   ```

2. Confirm the diagram satisfies the validation bar before you move on: the **end-to-end chain is shown across all five stages/layers**, in order, with the AI stage visible as its own step rather than folded into the identity or endpoint signals.

   {: .tip }
   > Keep the visualization deliberately simple — five nodes, four arrows, one timeline. Its persuasive power is the contrast: five quiet alerts in five different tools versus one obvious line when you connect them.

   {: .note }
   > 📷 **Screenshot to capture:** the finished cross-layer diagram, with all five stages and the connecting arrows visible end to end.

</details>

Congratulations! You've produced the centerpiece of the Proof Through Scenario: a Copilot-generated end-to-end summary and a cross-layer diagram that show the full five-stage chain — Identity → Data/AI → AI → Infrastructure → Endpoint — in a single, executive-readable view. Next, you'll pair this picture with a precise account of *where detection and response broke down*.
